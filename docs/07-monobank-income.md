# 07 — Дохід із monobank (Personal API)

Джерело доходу власника — **виключно monobank** (ПриватБанк використовується лише для КЕП-ключа,
грошей там немає). Тож для розрахунку доходу ФОП тягнемо виписку з monobank Personal API.

## Токен

- Персональний токен monobank лежить у `mono-token.txt` (у `.gitignore`).
- Тип — **тільки читання** (виписки + баланс). Рухати гроші ним НЕ можна.
- Заголовок запиту: `X-Token: <токен>`.
- Отримати новий: https://api.monobank.ua/ → підтвердження в застосунку.

## API

| Метод | Призначення |
|---|---|
| `GET /personal/client-info` | список рахунків (`id`, `iban`, `type`, `balance`, `currencyCode`) |
| `GET /personal/statement/{account}/{from}/{to}` | виписка (транзакції) за період; `from`/`to` — unix-секунди |

**Ліміти:** 1 запит на виписку **раз на 60 секунд**; діапазон одного запиту — **макс 31 день**.
`income.js` сам робить паузи 61с і ретраїть на 429. Квартал = 3 запити ≈ 2 хв.

**Формат транзакції:** `amount` — у копійках, у валюті рахунку, **знак +** = зарахування.
`operationAmount`+`currencyCode` — сума й валюта оригінальної операції (для валютних конвертацій).
`counterName` — контрагент.

## Рахунки власника (з `client-info`)

| Рахунок | id | Валюта | Роль |
|---|---|---|---|
| **ФОП USD** | `6P6YBDpEpANsdsnE11vSjw` | USD | **сюди приходить дохід** (SWIFT від клієнтів) |
| ФОП UAH | `WIjgghB5lP-pF87kbQUq0g` | UAH | лише власні конвертації з USD-рахунку |
| black UAH/USD, madeInUkraine, eAid | … | | особисті, не ФОП |

## ⚠️ Головне: «надходження» ≠ «дохід»

При підрахунку доходу ФОП **рахуються тільки зовнішні платежі від клієнтів**, а не будь-яке зарахування:

- **Дохід (оподатковується):** зовнішні SWIFT-платежі від клієнтів на **ФОП USD** рахунок
  (контрагент — іноземна компанія, напр. `Luca Labs As`, `Backdigital GmbH`).
- **НЕ дохід:** зарахування на **ФОП UAH** рахунок із контрагентом **«Кулуєв Андрій»** та поміткою
  «операція X USD» — це **власні перекази/конвертації** з USD-рахунку на гривневий (далі на картку).
  Переміщення між своїми рахунками податком не обкладається.

> Правило фільтра «чистого доходу»: брати `amount > 0` на **ФОП USD** рахунку, де `counterName`
> НЕ дорівнює власнику. Зарахування від самого себе — відкидати.

## Валюта і податкова база

Дохід приходить у **USD**. Для декларації він перераховується в **гривні за курсом НБУ на дату
надходження** кожного платежу. Тому «сума в UAH для податків» ≈ сума USD × курс НБУ того дня
(не поточний курс). Приблизна звірка Q1 2026: $6 790,48 × ~42,9 ≈ 291 342 грн = задеклароване.

## Використання (`Monobank::Client`)

```ruby
mono = Monobank::Client.new(token: File.read("../mono-token.txt").strip)
mono.fop_accounts                # ФОП-рахунки (id, iban, валюта)
usd = mono.fop_accounts.find { |a| a["currencyCode"] == 840 }["id"]

# надходження за квартал (пам'ятай: ліміт 1 запит/60с, діапазон ≤31 день — тягни помісячно)
mono.credits(account: usd, from: Time.zone.local(2026,4,1), to: Time.zone.local(2026,5,1))
```
