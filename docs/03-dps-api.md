# 03 — Приватний API Електронного кабінету ДПС

Офіційні описи:
- Приватна частина (дані платника): https://cabinet.tax.gov.ua/help/api-registers-int.html
- Подання звітності (POST): https://cabinet.tax.gov.ua/help/api.html
- Відкрита частина (реєстри, без КЕП): https://cabinet.tax.gov.ua/help/api-registers.html

## Базові адреси

| Призначення | URL |
|---|---|
| Приватні дані (GET) | `https://cabinet.tax.gov.ua/ws/public_api` |
| Подання документів (POST) | `https://cabinet.tax.gov.ua/cabinet/public/api/exchange` |
| Тестовий сервер | `https://cabinet.tax.gov.ua:9443` |

## Авторизація (найважливіше)

Заголовок **`Authorization`** = **РНОКПП, підписаний внутрішнім (enveloping) підписом КЕП,
з доданим сертифікатом, у Base64**. Це не bearer/JWT і не OAuth-пароль — це **CMS/PKCS#7 signedData**
(тому значення починається з `MII…` — ознака ASN.1 SEQUENCE).

Як ми це робимо (`Dps::KepSigner` → `lib/kep/sign.js`, докладніше в `04-keys-and-signing.md`):
```js
// 🔴 БЕЗ time: — інакше ДПС відхиляє підпис ("хибний підпис"). Див. 04-keys-and-signing.md.
const der  = await box.pipe(Buffer.from('3525511937'), [{ op: 'sign' }], {});
const auth = Buffer.from(der).toString('base64');            // це і є значення Authorization
```
Заголовки запиту:
```
Content-Type: application/json
Accept: application/json
Authorization: <base64 CMS>
```

**Альтернатива без коду:** можна залогінитись у кабінет браузером (навіть хмарним підписом monobank),
відкрити DevTools → Network → скопіювати готовий заголовок `Authorization`. Він діє, поки жива сесія.
Ми ж генеруємо його самі файловим ключем — тому працює автономно.

**Ліміт:** ~1000 запитів на добу на токен (перевищення → токен відкликається). Кешуй у `out/`.

## Endpoint-и, які ВЖЕ використовуються (`check.js`)

Усі — `GET`, база `…/ws/public_api`:

| Endpoint | Що повертає |
|---|---|
| `/payer_card` | реєстраційна картка платника (ІПН, ДПІ, група ЄП, ЄСВ, КВЕД, ПДВ, РРО…) |
| `/ta/debt` | податковий борг; **`[]` = боргу немає** |
| `/ta/splatp?year=YYYY` | стан розрахунків з бюджетом за рік, по кожному податку (див. поля нижче) |
| `/reg_doc/list?periodYear=YYYY` | перелік поданих/зареєстрованих документів (декларації тощо) |
| `/post/sent?page=N` | вихідні документи (що надіслав платник) |
| `/post/incoming?page=N` | вхідні документи/листи від ДПС |

## Endpoint-и, відомі, але ще НЕ використані

| Endpoint | Що |
|---|---|
| `/ta/splatp/{year}/{cSti}/{idplt}/pa?page=N` | деталізація операцій по конкретному рахунку |
| `/reg_doc/doc/{year}/{codRegdoc}/pdf` \| `/xml` | конкретна декларація у PDF/XML |
| `/reg_doc/docs/{year}/{codRegdoc}` | документ з додатками |
| `/post/sent/{year}/{id}/pdf`, `/incoming/{year}/{id}/pdf`\|`/xml` | PDF/XML листування |
| `/payer_card/{group}?page=&size=` | окремий блок картки (напр. `14` — банківські рахунки) |
| POST `…/exchange/report` | надіслати один підписаний звіт (`contentBase64`) |
| POST `…/exchange/reportzip` | надіслати архів звітів (`zipBase64`) |
| POST `…/exchange/kvt_by_id` | отримати квитанції по звіту (`encryptedId`) |

## Розшифровка полів `/ta/splatp` (стан розрахунків)

Кожен елемент масиву = окремий податок/рахунок за рік:

| Поле | Значення |
|---|---|
| `namePlt` | назва податку (ЄДИНИЙ ПОДАТОК / ВІЙСЬКОВИЙ ЗБІР / ЄДИН.ВНЕС…) |
| `shot` | код бюджетної класифікації (ЄП=18050400, ВЗ=11011700, ЄСВ=71040000) |
| `budgetAccountIban` | **IBAN, куди платити** цей податок цьогоріч |
| `narah0` / `narahEnd` | нараховано (початок/кінець періоду) |
| `splbd0` / `splbdEnd` | сплачено |
| `perepl0` / `pereplEnd` | **переплата** (надміру сплачено) |
| `nedoim0` / `nedoimEnd` | **недоїмка** (недоплата) |
| `penia0` / `peniaEnd` | **пеня** |
| `debtAll` / `debtAllEnd` | **сукупний борг** (0 = боргу немає) |
| `paymentEnabled` | чи дозволена онлайн-оплата з кабінету |
| `cSti`, `idplt` | коди для деталізації (`/ta/splatp/.../pa`) |

**Як читати "чи все ок":** борг = `debtAll==0` І `nedoimEnd==0` І `peniaEnd==0` по всіх рядках,
а `/ta/debt` порожній. Якщо `narahEnd > splbdEnd`, але `debtAll==0` — різниця або ще не настала строком,
або **покрита переплатою минулого року** (перевір `pereplEnd` попереднього року).

## Розшифровка полів `/reg_doc/list` (подані документи)

| Поле | Значення |
|---|---|
| `doc` | код форми (напр. `F0103309` — декларація ЄП 3 групи) |
| `docName` | людська назва документа |
| `period` | період (напр. «I квартал 2026», «Пiврiччя 2025») |
| `dget` | коли ДПС отримала документ |
| `dterm` | граничний строк подачі (дедлайн) |
| `nreg` | реєстраційний номер |

**Логіка "подано вчасно":** документ присутній у списку (=зареєстрований/прийнятий) і `dget <= dterm`.
